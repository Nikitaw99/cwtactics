 15 /**
 16  * The model layer holds all necessary data for a game round. This layer can be
 17  * extended to store additional data for game rounds.
 18  * </br></br>
 19  * If you extend this layer you should follow two rules. At first remember that
 20  * every property of this layer will be saved in a save game. The current
 21  * persistence layer implementation uses a json algorithm to serialize all model
 22  * data. This means you cannot store cyclic data structures in the model layer.
 23  * Furthermore you should not place functions in this layer because this would
 24  * not follow the specification of this layer.
 25  *
 26  * @example
 27  *  usage -> read only for client
 28  *
 29  * @namespace

 48 /**
 49  * This is the main access layer for the custom wars tactics game client. All
 50  * data changing actions will be invoked from this layer.
 51  *
 52  * The layer itself is build as state machine which represents a player action.
 53  * Every action starts by a selection of a tile. Which the selected object will
 54  * be choosen by the state of the tile. An empty tile leads to a map action. An
 55  * empty (owned) property leads to a property actions like buying an unit. The
 56  * last option will be choosen if the tile is occupied by an own unit.
 57  *
 58  * Action type search:
 59  * <ul>
 60  *   <li>1. tile occupied by an actable unit</br>
 61  *        if yes then unit action else 2.</li>
 62  *   <li>2. is tile a property that can build things</br>
 63  *        if yes then property action else 3.</li>
 64  *   <li>3. map action</li>
 65  * </ul>
 66  *
 67  * @namespace
 68  */
 69 var controller  = {};
