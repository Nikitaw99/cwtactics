Fog System - Milestone 2.3
the game holds a matrix with the vision values
if a mover will be placed on the map every tile in range gets +1
if a mover will be removed from the map every tile in range gets –1
a tile is visible if the fog value greater than 0
fog changes if a property is captured
fog changes if an unit moves from a to b
fog changes if a unit will be placed on the map
fog changes if the turn starts
the game holds the last active local player instance
not every player will be playable (with AI mechanisms in milestone 4)
only the fog map of the last active player will be shown
Co System
a player has a main co
a player has a side co
a player have a power meter bar
after a battle the attacker gets power meter points for the damage dealt
after a battle the defender gets power meter points for the damage he got
the power bar has a lower power part and a higher level part
a player has an active co
a player can active normal co power
a player can active super co power
a player can active tag co power
a co power effect remains for one in-game day
Tag Engine
tag script format which is JSON friendly
tag scripts can be global for all players
tag scripts can be player only and only activates for that players
tag scripts can be active for the whole gam round
tag scripts can be active for a given number of dates
a tag event will be triggered from source code
a unit position ( tile type ) can be set as condition
unit fire types can be set as condition
unit move types can be used as condition
unit type can be used as condition
